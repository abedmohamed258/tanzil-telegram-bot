# Implementation Plan: Tanzil Engine Core Extraction

**Branch**: `004-tanzil-engine-core` | **Date**: 2026-04-17 | **Spec**: [/specs/004-tanzil-engine-core/spec.md](spec.md)
**Input**: Feature specification from `/specs/004-tanzil-engine-core/spec.md`

## Summary

The primary goal is to extract the core Tanzil engine logic into a standalone, asynchronous component. This includes implementing an `asyncio`-based event bus, managing high-concurrency extractions (up to 50 tasks), and providing a CLI for engine management. The engine will maintain all state in-memory for this phase.

## Technical Context

**Language/Version**: Python 3.10+ (as per AGENTS.md)
**Primary Dependencies**: `typer`, `pyyaml`, `asyncio`
**Storage**: N/A (In-memory state only)
**Testing**: `pytest` (standard for Python projects)
**Target Platform**: Linux/Universal (Python 3.10+)
**Project Type**: CLI / Engine Core
**Performance Goals**: Initialization < 100ms, 50+ concurrent tasks
**Constraints**: < 100ms feedback loop (Constitution), structured JSON results
**Scale/Scope**: Phase 2 extraction core
